# Vote #002: Agent 배치 전략

> **발신**: 통제사
> **수신**: 정운 (ChatGPT)
> **안건**: 5개 이슈 집중공략을 위한 최적 Agent 편성

---

## 배경

3시간 내에 5개의 핵심 이슈를 해결해야 합니다.

### Agent 프로필

| Agent | 플랫폼 | 강점 | 약점 |
|:------|:-------|:-----|:-----|
| **정운** | ChatGPT | 빠른 초안, 브레인스토밍, 광범위한 지식 | 코드베이스 직접 접근 불가 |
| **권준** | Claude Code (CLI1) | 파일 읽기/쓰기, 코드 실행, Git 연동 | 컨텍스트 제한 |
| **나대용** | Claude Code (CLI2) | 파일 읽기/쓰기, 병렬 작업 가능 | 권준과 동일 툴셋 (중복) |
| **김완** | Claude (Web) | 긴 문서 검토, 전략적 조언, 품질 검수 | 파일 직접 수정 불가 |

---

## 두 가지 옵션

### **OPTION 1: 4인 체제** (정운 + 권준 + 나대용 + 김완)

```
정운 ──→ 초안/아이디어 생성
           ↓
권준 ──→ 이슈 #1, #2 구현
나대용 ──→ 이슈 #3, #4 구현  (병렬)
           ↓
김완 ──→ 품질 검수 + 이슈 #5
```

**장점:**
- 권준/나대용 병렬 작업으로 처리량 2배
- 분업 명확: 생성(정운) → 구현(권준+나대용) → 검수(김완)
- 3시간 내 5개 이슈 완료 가능성 높음

**단점:**
- 권준과 나대용이 동일 툴셋 → 조율 비용 발생
- 같은 파일 동시 수정 시 충돌 가능
- 통제사 조율 부담 증가 (4개 창 관리)

---

### **OPTION 2: 3인 체제** (정운 + 권준 + 김완)

```
정운 ──→ 초안/아이디어 + 이슈 #5 검토
           ↓
권준 ──→ 이슈 #1~#4 순차 구현
           ↓
김완 ──→ 품질 검수 + 전략 조언
```

**장점:**
- 조율 간단: 3개 창만 관리
- 파일 충돌 없음 (권준만 파일 수정)
- 역할 중복 없음

**단점:**
- 권준 혼자 4개 이슈 → 병목 가능
- 3시간 내 5개 완료 불확실
- 권준 컨텍스트 초과 시 전체 지연
